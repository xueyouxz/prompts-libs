# 需求分析与拆解

> **分类**：`prompts/analysis/`
> **适用场景**：将模糊的业务需求拆解为清晰的产品需求、用户故事和验收标准。
> **版本**：1.0

---

## 描述

提供原始业务需求描述，AI 将帮助澄清需求、识别边界条件、拆解用户故事，并生成可直接录入需求管理工具的内容。

## 使用方法

1. 复制下方模板到 AI 对话框。
2. 填入 `{{原始需求}}` 和 `{{背景信息}}`。

## Prompt 模板

```
请帮我对以下业务需求进行分析和拆解。

## 原始需求

{{原始需求}}

## 背景信息

- **产品**：{{产品名称}}
- **用户角色**：{{用户角色}}
- **业务目标**：{{业务目标}}

## 分析要求

1. **需求澄清**：列出需要向业务方确认的问题（如有歧义）
2. **范围边界**：明确本次需求的 In Scope 和 Out of Scope
3. **用户故事拆解**：按"作为 [角色]，我希望 [功能]，以便 [价值]"格式拆解
4. **验收标准**：为每个用户故事写出具体的验收标准（Given-When-Then 格式）
5. **风险识别**：识别潜在的技术风险或依赖

## 输出格式

请按以下结构输出 Markdown 内容：

### 需求澄清问题
### 需求范围
### 用户故事列表
### 验收标准
### 风险与依赖
```

## 变量说明

| 变量 | 说明 | 示例 |
|------|------|------|
| `{{原始需求}}` | 未经加工的业务需求描述 | `用户希望能在手机上查看订单状态` |
| `{{产品名称}}` | 所在产品或系统名 | `电商 App` |
| `{{用户角色}}` | 主要用户类型 | `已登录的买家` |
| `{{业务目标}}` | 该需求要实现的业务价值 | `减少客服咨询量，提升用户满意度` |
