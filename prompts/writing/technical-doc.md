# 技术文档撰写

> **分类**：`prompts/writing/`
> **适用场景**：为 API、功能模块、系统架构或操作流程撰写清晰、标准化的技术文档。
> **版本**：1.0

---

## 描述

提供技术内容概述，AI 将生成结构完整、符合技术写作规范的文档草稿，包括概述、使用说明、参数说明和示例。

## 使用方法

1. 复制下方模板到 AI 对话框。
2. 填入 `{{文档主题}}`、`{{目标读者}}` 和 `{{技术内容描述}}`。

## Prompt 模板

```
请帮我撰写以下技术文档。

## 文档信息

- **文档主题**：{{文档主题}}
- **目标读者**：{{目标读者}}（如：使用该 API 的后端开发者）
- **文档类型**：{{文档类型}}（如：API 参考、快速入门、操作手册）

## 技术内容描述

{{技术内容描述}}

## 写作要求

1. 使用 Markdown 格式
2. 结构包含：概述 → 前提条件 → 快速开始 → 详细说明 → 示例 → 常见问题
3. 所有代码块指定语言（如 ```json、```bash）
4. 参数/字段使用表格呈现（名称、类型、是否必填、说明、默认值）
5. 语言简洁精准，避免口语化表达

## 输出格式

请输出完整的 Markdown 格式文档。
```

## 变量说明

| 变量 | 说明 | 示例 |
|------|------|------|
| `{{文档主题}}` | 文档描述的技术主题 | `用户认证 REST API` |
| `{{目标读者}}` | 阅读文档的受众 | `集成该 API 的第三方开发者` |
| `{{文档类型}}` | 文档的类型 | `API 参考文档` |
| `{{技术内容描述}}` | 需要记录的技术内容要点 | `POST /auth/login 接口，接收邮箱和密码，返回 JWT token` |
