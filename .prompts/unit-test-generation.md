# 单元测试生成

> **分类**：`.prompts/`
> **适用场景**：为已有函数或模块自动生成完整的单元测试用例，覆盖正常路径、边界条件和异常场景。
> **版本**：1.0

---

## 描述

输入目标函数代码，AI 将生成覆盖率高、可直接运行的单元测试，包含正常用例、边界用例和异常用例。

## 使用方法

1. 复制下方模板到 AI 对话框。
2. 填入 `{{编程语言}}`、`{{测试框架}}`、`{{代码片段}}`。
3. 按需说明 `{{特殊要求}}`（如覆盖率目标、Mock 需求等）。

## Prompt 模板

```
请为以下 {{编程语言}} 代码使用 {{测试框架}} 生成全面的单元测试。

## 待测试代码

```{{编程语言}}
{{代码片段}}
```

## 测试要求

1. 覆盖以下场景：
   - 正常路径（Happy Path）
   - 边界条件（空值、零值、最大值等）
   - 异常场景（无效输入、异常抛出等）
2. 每个测试用例需有清晰的描述（`describe` / `it` / `test` 的命名）
3. 使用 Mock / Stub 隔离外部依赖
4. {{特殊要求}}

## 输出格式

请先列出测试用例设计表（场景、输入、期望输出），然后给出完整可运行的测试代码。
```

## 变量说明

| 变量 | 说明 | 示例 |
|------|------|------|
| `{{编程语言}}` | 目标语言 | `TypeScript`、`Python` |
| `{{测试框架}}` | 使用的测试框架 | `Jest`、`pytest`、`JUnit 5` |
| `{{代码片段}}` | 需要测试的函数或类 | *(粘贴代码)* |
| `{{特殊要求}}` | 额外要求（可选） | `需要 Mock axios 请求` |
