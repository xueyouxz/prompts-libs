# 代码审查

> **分类**：`.prompts/`
> **适用场景**：对已有代码进行全面的质量审查，发现潜在问题并给出改进建议。
> **版本**：1.0

---

## 描述

请 AI 扮演高级工程师对代码进行 Code Review，覆盖可读性、性能、安全性、可维护性等维度，并给出具体的修改建议。

## 使用方法

1. 复制下方模板到 AI 对话框。
2. 将 `{{编程语言}}` 替换为实际语言，`{{代码片段}}` 替换为要审查的代码。
3. 可选：补充 `{{上下文说明}}` 来描述代码的业务背景。

## Prompt 模板

```
你是一位经验丰富的高级软件工程师，请对以下 {{编程语言}} 代码进行全面的代码审查。

## 代码

```{{编程语言}}
{{代码片段}}
```

## 上下文

{{上下文说明}}

## 审查要求

请从以下维度进行分析，并对每个问题给出具体的修改建议（包含修改后的代码示例）：

1. **正确性**：逻辑是否正确，有无明显 bug
2. **可读性**：命名、注释、代码结构是否清晰
3. **性能**：有无明显的性能瓶颈或低效操作
4. **安全性**：有无安全漏洞（如注入、越界、敏感信息泄漏等）
5. **可维护性**：是否遵循 SOLID 原则，是否易于扩展
6. **测试覆盖**：是否便于单元测试，有无边界条件未处理

请以表格汇总所有问题，并按严重程度（严重 / 建议 / 优化）标注。
```

## 变量说明

| 变量 | 说明 | 示例 |
|------|------|------|
| `{{编程语言}}` | 代码所用的编程语言 | `Python`、`TypeScript` |
| `{{代码片段}}` | 需要审查的完整代码 | *(粘贴代码)* |
| `{{上下文说明}}` | 代码的业务背景（可选） | `这是一个处理用户登录的函数` |

## 使用示例

**输入**：

```
你是一位经验丰富的高级软件工程师，请对以下 Python 代码进行全面的代码审查。

## 代码

```python
def get_user(id):
    conn = db.connect()
    result = conn.execute("SELECT * FROM users WHERE id=" + id)
    return result.fetchone()
```

## 上下文

用于根据用户 ID 查询用户信息的数据库访问层函数。

## 审查要求
...
```

**输出示例**：

```
| 问题 | 维度 | 严重程度 | 修改建议 |
|------|------|---------|---------|
| SQL 拼接导致注入漏洞 | 安全性 | 严重 | 使用参数化查询 `conn.execute("... WHERE id=?", (id,))` |
| 数据库连接未关闭 | 可维护性 | 严重 | 使用 `with` 语句或 `try/finally` 确保连接关闭 |
| 函数参数无类型注解 | 可读性 | 优化 | 添加 `def get_user(id: int) -> Optional[dict]:` |
```
